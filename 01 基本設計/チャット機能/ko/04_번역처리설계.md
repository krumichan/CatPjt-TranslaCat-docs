# 번역 처리 설계

## 1. 기본 방침

- 메시지 전송과 번역 처리는 분리한다.
- 메시지는 즉시 저장하고 WebSocket으로 전송한다.
- 번역은 이후 비동기로 실행한다.
- 번역 실패는 메시지 전송 실패로 처리하지 않는다.

## 2. 처리 흐름

```text
사용자 메시지 전송
↓
BE 메시지 원문 저장
↓
WebSocket message.created 전송
↓
채팅방 멤버들의 언어 설정 조회
↓
필요한 번역 언어 목록 계산
↓
chat_message_translation PENDING 생성
↓
비동기 번역 실행
↓
번역 성공 시 COMPLETED 저장
↓
WebSocket translation.completed 전송
```

## 3. 번역 대상 언어 계산

번역 대상 언어는 아래 값을 기준으로 계산한다.

- 채팅방 멤버의 `receive_translation_language_code`
- 발신자의 `study_language_code`

단, 아래 경우는 제외한다.

- 번역 대상 언어가 원문 언어와 동일한 경우
- 동일 messageId + languageCode 번역이 이미 존재하는 경우

## 4. 상태 정의

| 상태 | 설명 |
|---|---|
| PENDING | 번역 대기 중 |
| COMPLETED | 번역 완료 |
| FAILED | 번역 실패 |

## 5. 실패 처리

- 실패 시 status를 FAILED로 저장한다.
- error_message에 실패 사유를 저장한다.
- FE에는 translation.failed 이벤트를 보낼 수 있다.
- 사용자는 원문 메시지를 계속 볼 수 있어야 한다.

## 6. 중복 방지

`chat_message_translation`에는 아래 unique index를 둔다.

```sql
CREATE UNIQUE INDEX uk_message_translation_message_language
ON chat_message_translation (message_id, language_code);
```
