# 채팅 기능 DB 설계

## 1. 문서 목적

본 문서는 TranslaCat 채팅 기능 Phase 1 개발에 필요한 DB 구조를 정의한다.

Phase 1에서는 사람끼리 이용하는 실시간 다국어 번역 채팅 MVP를 대상으로 하며, 채팅방, 채팅방 멤버, 메시지 원문, 메시지 번역 결과를 MySQL에 저장한다.

또한 Phase 1.5의 친구 기반 채팅과 Phase 2의 오픈 채팅 / AI 채팅 확장을 고려하여 `source_type`, `sender_type = SYSTEM` 구조를 반영한다.

## 2. 설계 방침

* Phase 1에서는 MySQL을 주 데이터베이스로 사용한다.
* 메시지 원문과 번역 결과는 모두 MySQL에 저장한다.
* 채팅 이력 조회는 최신 메시지 기준으로 수행한다.
* 과거 메시지 조회는 cursor 기반으로 수행한다.
* 같은 메시지와 같은 언어의 번역은 중복 생성하지 않는다.
* 채팅방 멤버별로 원문 기준 언어, 수신 번역 언어, 원문/번역 표시 여부를 관리한다.
* Phase 1에서는 `source_type = MANUAL`을 기본값으로 사용한다.
* Phase 1.5의 친구 채팅은 `source_type = FRIEND`로 확장한다.
* Phase 2의 오픈 채팅 및 AI 채팅은 `source_type = OPEN`, `source_type = AI`로 확장한다.
* AI 메시지와 시스템 메시지를 구분하기 위해 `sender_type = SYSTEM`을 별도로 정의한다.

| 테이블                      |   Phase | 설명                          |
| ------------------------ | ------: | --------------------------- |
| chat_room                | Phase 1 | 채팅방 기본 정보                   |
| chat_room_member         | Phase 1 | 채팅방 참여자 및 사용자별 언어 설정        |
| chat_message             | Phase 1 | 채팅 메시지 원문                   |
| chat_message_translation | Phase 1 | 메시지별 번역 결과                  |
| chat_ai_member           | Phase 2 | AI 채팅 멤버. Phase 2 대상        |
| open_chat_profile        | Phase 2 | 오픈 채팅 전용 가상 프로필. Phase 2 대상 |

## 4. chat_room

채팅방 기본 정보를 관리한다.

Phase 1에서는 1:1 채팅방과 그룹 채팅방을 생성한다. 오픈 채팅은 Phase 2 대상이지만, Phase 1.5 / Phase 2 확장 시 기존 DIRECT 방이 충돌하지 않도록 `source_type`을 함께 관리한다.

| 컬럼          | 타입            | 필수 | 설명                                  |
| ----------- | ------------- | -: | ----------------------------------- |
| id          | BIGINT PK     |  Y | 채팅방 ID                              |
| room_type   | VARCHAR(30)   |  Y | 채팅방 타입. DIRECT, GROUP, OPEN         |
| source_type | VARCHAR(30)   |  Y | 채팅방 생성 출처. MANUAL, FRIEND, OPEN, AI |
| name        | VARCHAR(100)  |  N | 채팅방 이름. DIRECT의 경우 null 가능          |
| description | VARCHAR(500)  |  N | 채팅방 설명                              |
| owner_id    | BIGINT FK     |  N | 채팅방 생성자/소유자 user ID                 |
| active      | BOOLEAN       |  Y | 활성 여부                               |
| created_at  | DATETIME      |  Y | 생성일시                                |
| updated_at  | DATETIME      |  Y | 수정일시                                |
| deleted_at  | DATETIME NULL |  N | 삭제일시                                |

### room_type

| 값      | 설명                      |
| ------ | ----------------------- |
| DIRECT | 1:1 채팅방                 |
| GROUP  | 그룹 채팅방                  |
| OPEN   | 오픈 채팅방. Phase 2 확장용 예약값 |

### source_type

| 값      | 설명                        |
| ------ | ------------------------- |
| MANUAL | Phase 1 기본 생성 채팅방         |
| FRIEND | Phase 1.5 친구 목록에서 생성된 채팅방 |
| OPEN   | Phase 2 오픈 채팅에서 파생된 채팅방   |
| AI     | Phase 2 AI 채팅방            |

### 인덱스

```sql
CREATE INDEX idx_chat_room_owner_id
    ON chat_room (owner_id);

CREATE INDEX idx_chat_room_room_type
    ON chat_room (room_type);

CREATE INDEX idx_chat_room_source_type
    ON chat_room (source_type);
```

### 비고

* Phase 1 생성 API는 기본적으로 `source_type = MANUAL`을 사용한다.
* 친구 1:1 채팅과 오픈 1:1 채팅이 같은 DIRECT 방으로 재사용되지 않도록, DIRECT 방 조회 시 `room_type`과 `source_type`을 함께 고려한다.
* `memberCount`는 `chat_room` 컬럼이 아니라 `chat_room_member`의 active 멤버 수를 계산한 응답 값이다.

## 5. chat_room_member

채팅방 참여자와 사용자별 언어 설정을 관리한다.

| 컬럼                        | 타입            | 필수 | 설명                             |
| ------------------------- | ------------- | -: | ------------------------------ |
| id                        | BIGINT PK     |  Y | 채팅방 멤버 ID                      |
| chat_room_id              | BIGINT FK     |  Y | 채팅방 ID                         |
| user_id                   | BIGINT FK     |  Y | 사용자 ID                         |
| role                      | VARCHAR(30)   |  Y | 채팅방 내 역할. OWNER, ADMIN, MEMBER |
| original_language_code    | VARCHAR(10)   |  N | 사용자별 원문 기준 언어                  |
| translation_language_code | VARCHAR(10)   |  N | 사용자별 수신 번역 언어                  |
| show_original             | BOOLEAN       |  Y | 원문 표시 여부                       |
| show_translation          | BOOLEAN       |  Y | 번역 표시 여부                       |
| active                    | BOOLEAN       |  Y | 활성 멤버 여부                       |
| joined_at                 | DATETIME      |  Y | 참여일시                           |
| left_at                   | DATETIME NULL |  N | 퇴실일시                           |
| created_at                | DATETIME      |  Y | 생성일시                           |
| updated_at                | DATETIME      |  Y | 수정일시                           |
| deleted_at                | DATETIME NULL |  N | 삭제일시                           |

### role

| 값      | 설명      |
| ------ | ------- |
| OWNER  | 채팅방 소유자 |
| ADMIN  | 채팅방 관리자 |
| MEMBER | 일반 멤버   |

### 제약조건

```sql
ALTER TABLE chat_room_member
    ADD CONSTRAINT uk_chat_room_member_room_user
    UNIQUE (chat_room_id, user_id);
```

### 인덱스

```sql
CREATE INDEX idx_chat_room_member_user_active
    ON chat_room_member (user_id, active);

CREATE INDEX idx_chat_room_member_room_active
    ON chat_room_member (chat_room_id, active);
```

### 비고

* `original_language_code`와 `translation_language_code`가 null인 경우, 서비스 기본 언어 설정을 사용할 수 있다.
* `show_original`, `show_translation`은 사용자의 메시지 표시 기준으로 사용한다.
* `active = true`이고 `deleted_at is null`인 멤버를 현재 참여자로 본다.
* `memberCount`는 `active = true`이고 `deleted_at is null`인 멤버 수를 기준으로 계산한다.

## 6. chat_message

채팅 메시지 원문을 저장한다.

| 컬럼             | 타입             | 필수 | 설명                                 |
| -------------- | -------------- | -: | ---------------------------------- |
| id             | BIGINT PK      |  Y | 메시지 ID                             |
| chat_room_id   | BIGINT FK      |  Y | 채팅방 ID                             |
| sender_user_id | BIGINT FK NULL |  N | 사용자 발신자 ID. SYSTEM 메시지의 경우 null 가능 |
| sender_type    | VARCHAR(30)    |  Y | 발신자 타입. USER, AI, SYSTEM           |
| message_type   | VARCHAR(30)    |  Y | 메시지 타입. TEXT, SYSTEM               |
| content        | TEXT           |  Y | 메시지 원문 내용                          |
| status         | VARCHAR(30)    |  Y | 메시지 상태. SENT, DELETED              |
| created_at     | DATETIME       |  Y | 생성일시                               |
| updated_at     | DATETIME       |  Y | 수정일시                               |
| deleted_at     | DATETIME NULL  |  N | 삭제일시                               |

### sender_type

| 값      | 설명                           |
| ------ | ---------------------------- |
| USER   | 일반 사용자 발신 메시지                |
| AI     | AI가 실제로 발화한 메시지. Phase 2 확장용 |
| SYSTEM | 시스템 안내/공지성 메시지               |

### message_type

| 값      | 설명         |
| ------ | ---------- |
| TEXT   | 일반 텍스트 메시지 |
| SYSTEM | 시스템 메시지    |

### status

| 값       | 설명    |
| ------- | ----- |
| SENT    | 정상 전송 |
| DELETED | 삭제됨   |

### 인덱스

```sql
CREATE INDEX idx_chat_message_room_created_id
    ON chat_message (chat_room_id, created_at, id);

CREATE INDEX idx_chat_message_room_id_id
    ON chat_message (chat_room_id, id);
```

### 비고

* 일반 사용자 메시지는 `sender_type = USER`, `message_type = TEXT`를 사용한다.
* 시스템 메시지는 `sender_type = SYSTEM`, `message_type = SYSTEM`을 사용한다.
* AI가 실제로 발화하는 메시지는 Phase 2에서 `sender_type = AI`, `message_type = TEXT`로 확장한다.
* 메시지 삭제 시 `status = DELETED`와 `deleted_at`을 함께 사용한다.

## 7. chat_message_translation

메시지별 번역 결과를 저장한다.

| 컬럼                 | 타입            | 필수 | 설명                                |
| ------------------ | ------------- | -: | --------------------------------- |
| id                 | BIGINT PK     |  Y | 번역 ID                             |
| chat_message_id    | BIGINT FK     |  Y | 메시지 ID                            |
| language_code      | VARCHAR(10)   |  Y | 번역 언어 코드                          |
| translated_content | TEXT          |  N | 번역문                               |
| status             | VARCHAR(30)   |  Y | 번역 상태. PENDING, COMPLETED, FAILED |
| failure_reason     | VARCHAR(1000) |  N | 번역 실패 사유                          |
| completed_at       | DATETIME NULL |  N | 번역 완료일시                           |
| created_at         | DATETIME      |  Y | 생성일시                              |
| updated_at         | DATETIME      |  Y | 수정일시                              |
| deleted_at         | DATETIME NULL |  N | 삭제일시                              |

### status

| 값         | 설명         |
| --------- | ---------- |
| PENDING   | 번역 대기/진행 중 |
| COMPLETED | 번역 완료      |
| FAILED    | 번역 실패      |

### 제약조건

```sql
ALTER TABLE chat_message_translation
    ADD CONSTRAINT uk_chat_message_translation_message_language
    UNIQUE (chat_message_id, language_code);
```

### 인덱스

```sql
CREATE INDEX idx_chat_translation_message_id
    ON chat_message_translation (chat_message_id);

CREATE INDEX idx_chat_translation_status
    ON chat_message_translation (status);
```

### 비고

* 같은 메시지와 같은 언어의 번역 결과는 하나만 존재한다.
* 번역 전에는 `status = PENDING` 상태로 생성한다.
* 번역 성공 시 `translated_content`, `completed_at`을 갱신하고 `status = COMPLETED`로 변경한다.
* 번역 실패 시 `failure_reason`을 저장하고 `status = FAILED`로 변경한다.
* 재시도 시 `status = PENDING`, `failure_reason = null`, `completed_at = null`로 초기화한다.

## 8. cursor 기반 메시지 조회

초기 조회는 최신 메시지를 기준으로 조회한다.

```sql
SELECT *
FROM chat_message
WHERE chat_room_id = :chatRoomId
  AND deleted_at IS NULL
ORDER BY created_at DESC, id DESC
LIMIT :limit;
```

과거 메시지 조회는 현재 화면에서 가장 오래된 메시지 ID를 cursor로 사용한다.

```sql
SELECT *
FROM chat_message
WHERE chat_room_id = :chatRoomId
  AND id < :cursorId
  AND deleted_at IS NULL
ORDER BY id DESC
LIMIT :limit;
```

정렬 기준은 화면 표시 시 오래된 메시지가 위, 최신 메시지가 아래에 오도록 FE 또는 API 응답 단계에서 조정할 수 있다.

## 9. soft delete 방침

| 테이블                      | soft delete 기준                               |
| ------------------------ | -------------------------------------------- |

| chat_message             | `status = DELETED`, `deleted_at` 설정          |

## 11. 비고

Phase 1의 채팅방 생성 요청은 내부 사용자 ID 기반의 `memberUserIds`를 사용한다.

다만 서비스 화면에서 내부 DB ID를 직접 사용하는 방식은 장기적으로 적절하지 않으므로, Phase 1.5에서 publicId 기반 사용자 검색 및 초대 방식으로 전환한다.

Phase 1에서 추가한 `source_type`은 친구 1:1 채팅, 오픈 1:1 채팅, AI 1:1 채팅이 같은 DIRECT 방으로 충돌하지 않도록 하기 위한 확장 기반이다.
