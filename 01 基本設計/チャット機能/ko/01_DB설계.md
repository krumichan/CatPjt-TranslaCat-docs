# 채팅 기능 DB 설계

## 1. 설계 방침

- 1차 개발에서는 MySQL을 주 데이터베이스로 사용한다.
- 메시지 원문과 번역 결과는 모두 MySQL에 저장한다.
- 채팅 이력 조회는 최신 100개 기준으로 수행한다.
- 과거 메시지 조회는 cursor 기반으로 수행한다.
- 같은 메시지와 같은 언어의 번역은 중복 생성하지 않는다.

## 2. 테이블 목록

| 테이블 | 설명 |
|---|---|
| chat_room | 채팅방 기본 정보 |
| chat_room_member | 채팅방 참여자 및 언어 설정 |
| chat_message | 채팅 메시지 원문 |
| chat_message_translation | 메시지 번역 결과 |
| chat_ai_member | AI 채팅 멤버. 2차 개발 대상 |

## 3. chat_room

| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | BIGINT PK | 채팅방 ID |
| room_type | VARCHAR | DIRECT, GROUP, OPEN |
| name | VARCHAR | 채팅방 이름 |
| owner_user_id | BIGINT | 방 생성자/소유자 |
| open_visibility | VARCHAR | PRIVATE, PUBLIC. OPEN 채팅용 |
| created_at | DATETIME | 생성일시 |
| updated_at | DATETIME | 수정일시 |
| deleted_at | DATETIME NULL | 삭제일시 |

## 4. chat_room_member

| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | BIGINT PK | 멤버 ID |
| room_id | BIGINT FK | 채팅방 ID |
| user_id | BIGINT FK | 사용자 ID |
| role | VARCHAR | OWNER, ADMIN, MEMBER |
| study_language_code | VARCHAR | 내가 쓴 메시지 하단에 표시할 학습 언어 |
| receive_translation_language_code | VARCHAR | 상대 메시지를 읽을 번역 언어 |
| joined_at | DATETIME | 참여일시 |
| left_at | DATETIME NULL | 퇴실일시 |

### 권장 인덱스

```sql
CREATE INDEX idx_chat_room_member_room_user
ON chat_room_member (room_id, user_id);

CREATE INDEX idx_chat_room_member_user
ON chat_room_member (user_id);
```

## 5. chat_message

| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | BIGINT PK | 메시지 ID |
| room_id | BIGINT FK | 채팅방 ID |
| sender_type | VARCHAR | USER, AI |
| sender_user_id | BIGINT NULL | 사용자 발신자 ID |
| sender_ai_member_id | BIGINT NULL | AI 발신자 ID. 2차 대상 |
| original_language_code | VARCHAR | 원문 언어 코드 |
| original_text | TEXT | 원문 메시지 |
| message_type | VARCHAR | TEXT, SYSTEM |
| status | VARCHAR | SENT, DELETED |
| created_at | DATETIME | 생성일시 |
| deleted_at | DATETIME NULL | 삭제일시 |

### 권장 인덱스

```sql
CREATE INDEX idx_chat_message_room_created_id
ON chat_message (room_id, created_at DESC, id DESC);
```

## 6. chat_message_translation

| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | BIGINT PK | 번역 ID |
| message_id | BIGINT FK | 메시지 ID |
| language_code | VARCHAR | 번역 언어 코드 |
| translated_text | TEXT | 번역문 |
| status | VARCHAR | PENDING, COMPLETED, FAILED |
| provider | VARCHAR | 번역 제공자 |

| created_at | DATETIME | 생성일시 |
| updated_at | DATETIME | 수정일시 |

### 권장 인덱스

```sql
CREATE UNIQUE INDEX uk_message_translation_message_language
ON chat_message_translation (message_id, language_code);

CREATE INDEX idx_message_translation_message
ON chat_message_translation (message_id);
```

## 8. cursor 기반 메시지 조회

초기 조회는 최신 100개를 가져온다.

```sql
SELECT *
FROM chat_message
WHERE room_id = :roomId
ORDER BY created_at DESC, id DESC
LIMIT 100;
```

과거 메시지 조회는 현재 화면에서 가장 오래된 메시지를 cursor로 사용한다.

```sql
SELECT *
FROM chat_message
WHERE room_id = :roomId
  AND (
    created_at < :oldestCreatedAt
    OR (created_at = :oldestCreatedAt AND id < :oldestMessageId)
  )
ORDER BY created_at DESC, id DESC
LIMIT 100;
```
