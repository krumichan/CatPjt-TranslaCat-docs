# チャット機能 DB設計

## 1. ドキュメント目的

本ドキュメントは、TranslaCatチャット機能 Phase 1 開発に必要なDB構造を定義する。

Phase 1では、人間同士で利用するリアルタイム多言語翻訳チャットMVPを対象とし、チャットルーム、チャットルームメンバー、メッセージ原文、メッセージ翻訳結果をMySQLに保存する。

また、Phase 1.5の友達ベースチャットおよびPhase 2のオープンチャット / AIチャット拡張を考慮し、`source_type`, `sender_type = SYSTEM` 構造を反映する。

## 2. 設計方針

* Phase 1ではMySQLを主データベースとして利用する。
* メッセージ原文と翻訳結果はすべてMySQLに保存する。
* チャット履歴取得は最新メッセージを基準に行う。
* 過去メッセージ取得はcursor方式で行う。
* 同一メッセージ・同一言語の翻訳は重複作成しない。
* チャットルームメンバーごとに原文基準言語、受信翻訳言語、原文/翻訳表示有無を管理する。
* Phase 1では `source_type = MANUAL` を基本値として利用する。
* Phase 1.5の友達チャットは `source_type = FRIEND` として拡張する。
* Phase 2のオープンチャットおよびAIチャットは `source_type = OPEN`, `source_type = AI` として拡張する。
* AIメッセージとシステムメッセージを区別するため、`sender_type = SYSTEM` を別途定義する。

## 4. chat_room

チャットルーム基本情報を管理する。

Phase 1では1:1チャットルームとグループチャットルームを作成する。オープンチャットはPhase 2対象だが、Phase 1.5 / Phase 2拡張時に既存DIRECTルームが衝突しないように `source_type` を併せて管理する。

| カラム         | 型             | 必須 | 説明                                  |
| ----------- | ------------- | -: | ----------------------------------- |
| id          | BIGINT PK     |  Y | チャットルームID                           |
| room_type   | VARCHAR(30)   |  Y | チャットルームタイプ。DIRECT, GROUP, OPEN      |
| source_type | VARCHAR(30)   |  Y | チャットルーム作成元。MANUAL, FRIEND, OPEN, AI |
| name        | VARCHAR(100)  |  N | チャットルーム名。DIRECTの場合はnull可            |
| description | VARCHAR(500)  |  N | チャットルーム説明                           |
| owner_id    | BIGINT FK     |  N | チャットルーム作成者/所有者user ID               |
| active      | BOOLEAN       |  Y | 有効状態                                |
| created_at  | DATETIME      |  Y | 作成日時                                |
| updated_at  | DATETIME      |  Y | 更新日時                                |
| deleted_at  | DATETIME NULL |  N | 削除日時                                |

### room_type

| 値      | 説明                        |
| ------ | ------------------------- |
| DIRECT | 1:1チャットルーム                |
| GROUP  | グループチャットルーム               |
| OPEN   | オープンチャットルーム。Phase 2拡張用予約値 |

### source_type

| 値      | 説明                            |
| ------ | ----------------------------- |
| MANUAL | Phase 1基本作成チャットルーム            |
| FRIEND | Phase 1.5の友達一覧から作成されたチャットルーム  |
| OPEN   | Phase 2のオープンチャットから派生したチャットルーム |
| AI     | Phase 2のAIチャットルーム             |

### インデックス

```sql
CREATE INDEX idx_chat_room_owner_id
    ON chat_room (owner_id);

CREATE INDEX idx_chat_room_room_type
    ON chat_room (room_type);

CREATE INDEX idx_chat_room_source_type
    ON chat_room (source_type);
```

### 備考

* Phase 1作成APIは基本的に `source_type = MANUAL` を利用する。
* 友達1:1チャットとオープン1:1チャットが同一DIRECTルームとして再利用されないよう、DIRECTルーム取得時は `room_type` と `source_type` を併せて考慮する。
* `memberCount` は `chat_room` カラムではなく、`chat_room_member` のactiveメンバー数を計算したレスポンス値である。

## 5. chat_room_member

チャットルーム参加者とユーザー別言語設定を管理する。

| カラム                       | 型             | 必須 | 説明                               |
| ------------------------- | ------------- | -: | -------------------------------- |
| id                        | BIGINT PK     |  Y | チャットルームメンバーID                    |
| chat_room_id              | BIGINT FK     |  Y | チャットルームID                        |
| user_id                   | BIGINT FK     |  Y | ユーザーID                           |
| role                      | VARCHAR(30)   |  Y | チャットルーム内ロール。OWNER, ADMIN, MEMBER |
| original_language_code    | VARCHAR(10)   |  N | ユーザー別原文基準言語                      |
| translation_language_code | VARCHAR(10)   |  N | ユーザー別受信翻訳言語                      |
| show_original             | BOOLEAN       |  Y | 原文表示有無                           |
| show_translation          | BOOLEAN       |  Y | 翻訳表示有無                           |
| active                    | BOOLEAN       |  Y | 有効メンバー状態                         |
| joined_at                 | DATETIME      |  Y | 参加日時                             |
| left_at                   | DATETIME NULL |  N | 退出日時                             |
| created_at                | DATETIME      |  Y | 作成日時                             |
| updated_at                | DATETIME      |  Y | 更新日時                             |
| deleted_at                | DATETIME NULL |  N | 削除日時                             |

### role

| 値      | 説明         |
| ------ | ---------- |
| OWNER  | チャットルーム所有者 |
| ADMIN  | チャットルーム管理者 |
| MEMBER | 一般メンバー     |

### 制約条件

```sql
ALTER TABLE chat_room_member
    ADD CONSTRAINT uk_chat_room_member_room_user
    UNIQUE (chat_room_id, user_id);
```

### インデックス

```sql
CREATE INDEX idx_chat_room_member_user_active
    ON chat_room_member (user_id, active);

CREATE INDEX idx_chat_room_member_room_active
    ON chat_room_member (chat_room_id, active);
```

### 備考

* `original_language_code` と `translation_language_code` がnullの場合、サービス基本言語設定を利用できる。
* `show_original`, `show_translation` はユーザーのメッセージ表示基準として利用する。
* `active = true` かつ `deleted_at is null` のメンバーを現在参加者とみなす。
* `memberCount` は `active = true` かつ `deleted_at is null` のメンバー数を基準に計算する。

## 6. chat_message

チャットメッセージ原文を保存する。

| カラム            | 型              | 必須 | 説明                             |
| -------------- | -------------- | -: | ------------------------------ |
| id             | BIGINT PK      |  Y | メッセージID                        |
| chat_room_id   | BIGINT FK      |  Y | チャットルームID                      |
| sender_user_id | BIGINT FK NULL |  N | ユーザー送信者ID。SYSTEMメッセージの場合はnull可 |
| sender_type    | VARCHAR(30)    |  Y | 送信者タイプ。USER, AI, SYSTEM        |
| message_type   | VARCHAR(30)    |  Y | メッセージタイプ。TEXT, SYSTEM          |
| content        | TEXT           |  Y | メッセージ原文内容                      |
| status         | VARCHAR(30)    |  Y | メッセージ状態。SENT, DELETED          |
| created_at     | DATETIME       |  Y | 作成日時                           |
| updated_at     | DATETIME       |  Y | 更新日時                           |
| deleted_at     | DATETIME NULL  |  N | 削除日時                           |

### sender_type

| 値      | 説明                         |
| ------ | -------------------------- |
| USER   | 一般ユーザー送信メッセージ              |
| AI     | AIが実際に発話したメッセージ。Phase 2拡張用 |
| SYSTEM | システム案内/通知メッセージ             |

### message_type

| 値      | 説明          |
| ------ | ----------- |
| TEXT   | 一般テキストメッセージ |
| SYSTEM | システムメッセージ   |

### status

| 値       | 説明   |
| ------- | ---- |
| SENT    | 正常送信 |
| DELETED | 削除済み |

### インデックス

```sql
CREATE INDEX idx_chat_message_room_created_id
    ON chat_message (chat_room_id, created_at, id);

CREATE INDEX idx_chat_message_room_id_id
    ON chat_message (chat_room_id, id);
```

### 備考

* 一般ユーザーメッセージは `sender_type = USER`, `message_type = TEXT` を利用する。
* システムメッセージは `sender_type = SYSTEM`, `message_type = SYSTEM` を利用する。
* AIが実際に発話するメッセージはPhase 2で `sender_type = AI`, `message_type = TEXT` として拡張する。
* メッセージ削除時は `status = DELETED` と `deleted_at` を併用する。

## 7. chat_message_translation

メッセージ別翻訳結果を保存する。

| カラム                | 型             | 必須 | 説明                              |
| ------------------ | ------------- | -: | ------------------------------- |
| id                 | BIGINT PK     |  Y | 翻訳ID                            |
| chat_message_id    | BIGINT FK     |  Y | メッセージID                         |
| language_code      | VARCHAR(10)   |  Y | 翻訳言語コード                         |
| translated_content | TEXT          |  N | 翻訳文                             |
| status             | VARCHAR(30)   |  Y | 翻訳状態。PENDING, COMPLETED, FAILED |
| failure_reason     | VARCHAR(1000) |  N | 翻訳失敗理由                          |
| completed_at       | DATETIME NULL |  N | 翻訳完了日時                          |
| created_at         | DATETIME      |  Y | 作成日時                            |
| updated_at         | DATETIME      |  Y | 更新日時                            |
| deleted_at         | DATETIME NULL |  N | 削除日時                            |

### status

| 値         | 説明       |
| --------- | -------- |
| PENDING   | 翻訳待機/進行中 |
| COMPLETED | 翻訳完了     |
| FAILED    | 翻訳失敗     |

### 制約条件

```sql
ALTER TABLE chat_message_translation
    ADD CONSTRAINT uk_chat_message_translation_message_language
    UNIQUE (chat_message_id, language_code);
```

### インデックス

```sql
CREATE INDEX idx_chat_translation_message_id
    ON chat_message_translation (chat_message_id);

CREATE INDEX idx_chat_translation_status
    ON chat_message_translation (status);
```

### 備考

* 同一メッセージ・同一言語の翻訳結果は1件のみ存在する。
* 翻訳前は `status = PENDING` 状態で作成する。
* 翻訳成功時は `translated_content`, `completed_at` を更新し、`status = COMPLETED` に変更する。
* 翻訳失敗時は `failure_reason` を保存し、`status = FAILED` に変更する。
* 再試行時は `status = PENDING`, `failure_reason = null`, `completed_at = null` に初期化する。

## 8. cursor方式メッセージ取得

初期取得は最新メッセージを基準に取得する。

```sql
SELECT *
FROM chat_message
WHERE chat_room_id = :chatRoomId
  AND deleted_at IS NULL
ORDER BY created_at DESC, id DESC
LIMIT :limit;
```

過去メッセージ取得は、現在画面で最も古いメッセージIDをcursorとして利用する。

```sql
SELECT *
FROM chat_message
WHERE chat_room_id = :chatRoomId
  AND id < :cursorId
  AND deleted_at IS NULL
ORDER BY id DESC
LIMIT :limit;
```

表示順は、古いメッセージが上、最新メッセージが下になるようにFEまたはAPIレスポンス段階で調整できる。

## 11. 備考

Phase 1のチャットルーム作成リクエストは、内部ユーザーIDベースの `memberUserIds` を利用する。

ただし、サービス画面で内部DB IDを直接利用する方式は長期的には適切ではないため、Phase 1.5でpublicIdベースのユーザー検索および招待方式へ移行する。

Phase 1で追加した `source_type` は、友達1:1チャット、オープン1:1チャット、AI 1:1チャットが同一DIRECTルームとして衝突しないようにするための拡張基盤である。
