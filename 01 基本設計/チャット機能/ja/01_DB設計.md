# チャット機能 DB設計

## 1. 設計方針

- 1次開発ではMySQLを主データベースとして利用する。
- メッセージ原文と翻訳結果はすべてMySQLに保存する。
- チャット履歴取得は最新100件を基準とする。
- 過去メッセージ取得はcursor方式で行う。
- 同一メッセージ・同一言語の翻訳は重複作成しない。

## 2. テーブル一覧

## 8. cursor方式メッセージ取得

初期取得は最新100件を取得する。

```sql
SELECT *
FROM chat_message
WHERE room_id = :roomId
ORDER BY created_at DESC, id DESC
LIMIT 100;
```

過去メッセージ取得は、現在画面で最も古いメッセージをcursorとして利用する。

```sql
SELECT *
FROM chat_message
WHERE room_id = :roomId
  AND (
    created_at < :oldestCreatedAt
    OR (created_at = :oldestCreatedAt AND id < :oldestMessageId)
  )
ORDER BY created_at DESC, id DESC
LIMIT 100;
```
