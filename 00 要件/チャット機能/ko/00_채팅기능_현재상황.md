# TranslaCat 채팅 기능 현재상황

## 1. 개요

TranslaCat의 채팅 기능은 사용자가 자신의 언어로 메시지를 작성하면, 다른 참여자가 각자 설정한 언어로 번역을 확인할 수 있는 실시간 다국어 채팅 기능이다.

1차 개발에서는 사람끼리 사용하는 실시간 다국어 번역 채팅 MVP를 대상으로 한다. AI 채팅 멤버 기능은 2차 개발 대상으로 분리한다.

## 2. 현재 개발 방침

### 1차 개발 대상

- 1:1 채팅
- 그룹 채팅
- 메시지 원문 저장
- 사용자별 수신 번역 언어 설정
- 사용자별 학습 언어 설정
- WebSocket 기반 실시간 메시지 송수신
- 메시지 저장 후 비동기 번역 처리
- 번역 완료 후 메시지 하단에 번역 표시
- 채팅방 최초 진입 시 최신 100개 메시지 표시
- 위로 스크롤 시 과거 메시지 100개 단위 추가 로딩

### 2차 개발 대상

- 오픈 채팅
- AI 채팅 멤버
- 방당 AI 채팅 멤버 최대 3명
- AI 자발 발언
- 읽음 처리
- 미읽음 수 표시
- 온라인 상태 표시
- Redis 도입 검토
- 알림 기능

## 3. 아키텍처 방침

### Backend

- Spring Boot 기반으로 구현한다.
- MySQL을 주 데이터베이스로 사용한다.
- 채팅 메시지 원문과 번역 결과는 MySQL에 영속화한다.
- WebSocket으로 실시간 메시지 송수신을 처리한다.
- 번역 처리는 메시지 저장과 분리하여 비동기로 실행한다.
- 동일 메시지와 동일 언어의 번역은 중복 생성하지 않는다.

### Frontend

- Next.js 기반으로 구현한다.
- 채팅 목록 화면은 `/chat`을 기본 경로로 한다.
- 채팅방 상세 화면은 `/chat/[roomId]`를 기본 경로로 한다.
- 초기 메시지 표시는 최신 100개로 제한한다.
- 과거 메시지는 cursor 기반으로 100개씩 추가 로딩한다.
- 신규 메시지와 번역 완료 이벤트는 WebSocket으로 수신한다.

## 7. 결정사항

- AI 채팅 멤버는 2차 개발 대상으로 한다.
- 1차 개발에서는 MySQL을 주 데이터베이스로 사용한다.
- NoSQL은 1차 개발에서 도입하지 않는다.
- Redis는 1.5차 또는 2차 이후 필요 시 검토한다.
- 채팅방 진입 시 최신 100개 메시지를 표시한다.
- 과거 메시지는 위 스크롤 시 100개씩 추가 로딩한다.
- 메시지 이력 조회는 offset 방식이 아니라 cursor 방식으로 구현한다.
- 메시지 저장과 번역 처리는 분리한다.
- 번역 실패가 발생하더라도 원문 메시지 송수신은 성공으로 처리한다.
