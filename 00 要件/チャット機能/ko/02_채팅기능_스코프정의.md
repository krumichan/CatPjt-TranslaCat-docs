# 채팅 기능 스코프 정의

## 1. 문서 목적

본 문서는 TranslaCat 채팅 기능의 Phase 1 / Phase 1.5 / Phase 2 개발 범위를 정의한다.

채팅 기능은 기본 메시지 송수신뿐만 아니라, 사용자별 번역 언어 설정, 비동기 AI 번역, 친구 기반 채팅, 오픈 채팅, AI 채팅 멤버, 오픈 채팅 전용 가상 프로필까지 확장될 수 있다.

따라서 각 Phase의 개발 범위를 명확히 구분하여, Phase 1 개발 중 불필요한 범위 확대를 방지하고, 이후 Phase 1.5 / Phase 2 확장 시 기존 구조를 크게 뒤엎지 않도록 하는 것을 목적으로 한다.

## 2. 전체 Phase 구분

| Phase     | 구분            | 목적                                 |
| --------- | ------------- | ---------------------------------- |
| Phase 1   | 기본 채팅 MVP     | 사람끼리 이용하는 실시간 다국어 번역 채팅 기능 구현      |
| Phase 1.5 | 친구 기반 소셜 채팅   | 친구 목록, 친구 추가, 프로필, 차단, 친구 채팅 기반 구축 |
| Phase 2   | 오픈 채팅 / AI 확장 | 오픈 채팅, AI 참여자, 가상 프로필 등 확장 기능 구현   |

## 3. Phase 1 개발 범위

Phase 1은 기본 채팅 MVP를 구현하는 단계로 정의한다.

### 3.1 채팅방

* 1:1 채팅방 생성
* 그룹 채팅방 생성
* 채팅방 목록 표시
* 채팅방 상세 조회
* 채팅방 참여자 수 표시
* 권한 없는 채팅방 접근 차단

### 3.2 메시지

* 메시지 원문 저장
* 메시지 목록 조회
* 최신 메시지 초기 표시
* 이전 메시지 추가 로딩
* WebSocket 기반 실시간 메시지 송수신
* WebSocket 재접속 시 REST 기반 최신 메시지 동기화
* 메시지 삭제 상태 표시 기반 준비

### 3.3 번역

* 메시지 발송 후 비동기 번역
* 메시지별 번역 결과 저장
* 번역 상태 관리

  * PENDING
  * COMPLETED
  * FAILED
* 번역 완료 이벤트 수신
* 번역 실패 이벤트 수신 가능 구조
* 메시지별 번역 표시
* 번역이 없는 경우 원문 유지
* 번역 실패 시 원문 유지

### 3.4 사용자별 언어 설정

* 사용자별 원문 기준 언어 설정
* 사용자별 수신 번역 언어 설정
* 사용자별 원문 표시 여부 설정
* 사용자별 번역 표시 여부 설정
* 설정 변경 후 메시지 번역 표시 기준 반영

### 3.5 Phase 1.5 / Phase 2 대비 구조 보정

Phase 1에서는 향후 확장을 위해 아래 구조를 반영한다.

* `ChatRoomSourceType` 도입

  * MANUAL
  * FRIEND
  * OPEN
  * AI
* Phase 1의 채팅방 생성은 기본적으로 `sourceType = MANUAL`을 사용한다
* Phase 1.5의 친구 채팅은 이후 `sourceType = FRIEND`로 확장한다
* Phase 2의 오픈 채팅 파생 채팅은 이후 `sourceType = OPEN`으로 확장한다
* Phase 2의 AI 채팅은 이후 `sourceType = AI`로 확장한다
* 시스템 메시지와 AI 메시지를 구분하기 위해 `senderType = SYSTEM`을 도입한다

## 4. Phase 1.5 개발 범위

Phase 1.5는 친구 기반 소셜 채팅 기능을 구축하는 단계로 정의한다.

### 4.1 사용자 프로필

* 공통 사용자 프로필
* 닉네임
* 프로필 이미지
* TranslaCat 전용 publicId
* publicId 기반 사용자 식별
* 자기 프로필 조회/수정

### 4.2 친구 기능

* publicId 기반 사용자 검색
* 친구 요청
* 친구 요청 수락
* 친구 요청 거절
* 친구 목록 조회
* 친구 삭제

### 4.3 차단 기초 기능

* 사용자 차단
* 차단 해제
* 차단한 사용자의 친구 요청 제한
* 차단한 사용자의 그룹 초대 제한

### 4.4 친구 1:1 채팅

* 친구 목록에서 1:1 채팅 시작
* 기존 친구 1:1 채팅방 재사용
* 친구 1:1 채팅방 중복 생성 방지

### 4.5 친구 그룹 채팅

* 친구 목록 기반 그룹 채팅 생성
* 친구 목록 기반 그룹 초대
* 비친구 초대 시 수락 기반 참가 검토

## 5. Phase 2 개발 범위

Phase 2는 오픈 채팅 및 AI 참여자를 포함하는 확장 단계로 정의한다.

### 5.1 오픈 채팅

* 오픈 1:1 채팅
* 오픈 그룹 채팅
* 오픈 채팅방 목록
* 오픈 채팅방 검색
* 오픈 채팅방 최대 인원 설정
* 오픈 채팅방 입장 정책
* 오픈 채팅방 운영 정책

### 5.2 AI 채팅

* AI와 1:1 채팅
* 오픈 그룹 채팅에 AI 초대
* 오픈 그룹 채팅 내 AI 최대 N명 제한
* AI 메시지와 사용자 메시지 구분
* AI 응답 제어

### 5.3 AI 참여 정책

오픈 채팅방은 방 단위로 AI 참여 정책을 가진다.

* AI 참여 불가
* AI 참여 가능 / AI 표시
* AI 참여 가능 / 개별 AI 표시 없음

AI 존재 가능성은 방 정보에 표시하고, 개별 AI 표시 여부는 방 정책에 따라 통일한다.

### 5.4 오픈 채팅 전용 가상 프로필

* 오픈 채팅방별 가상 프로필 생성
* 기본 프로필 또는 가상 프로필 선택 입장
* 오픈 채팅방별 닉네임 설정
* 오픈 채팅방별 프로필 이미지 설정
* 방 퇴장 시 가상 프로필 비활성화 검토

### 5.5 운영 및 확장 기능

* 신고
* 차단 고도화
* 강퇴
* 운영자/관리자 역할
* 읽음/미읽음 관리
* 온라인 상태 표시
* Redis 기반 scale-out 검토

## 6. Phase 1 제외 범위

아래 기능은 Phase 1 개발 범위에서 제외한다.

* 친구 기능
* 친구 요청/수락/거절
* 사용자 프로필 상세 관리
* publicId 기반 사용자 검색
* 차단 기능
* 오픈 채팅
* AI 채팅 멤버
* AI 자발 발화
* 오픈 채팅 전용 가상 프로필
* 신고/운영 정책
* 읽음/미읽음 관리
* 온라인 상태 표시
* Redis 기반 scale-out 구성

## 7. Phase 1 완료 기준

Phase 1은 아래 조건을 만족하면 완료로 판단한다.

* 채팅방 목록을 조회할 수 있다
* 채팅방을 생성할 수 있다
* 채팅방 참여자 수가 표시된다
* 채팅방 상세 화면에 진입할 수 있다
* 권한 없는 채팅방 접근이 차단된다
* 메시지를 전송할 수 있다
* WebSocket으로 메시지를 수신할 수 있다
* 초기 메시지 목록을 조회할 수 있다
* 이전 메시지를 추가 로딩할 수 있다
* 메시지 번역 결과를 표시할 수 있다
* 번역 완료 이벤트를 수신하고 화면에 반영할 수 있다
* 번역 실패 상태를 표시할 수 있다
* 사용자별 언어 설정을 변경할 수 있다
* 언어 설정 변경 후 번역 표시 기준이 반영된다
* FE 애플리케이션이 정상적으로 빌드된다
* BE 애플리케이션이 정상적으로 빌드된다

## 8. 비고

Phase 1의 채팅방 생성 요청은 내부 사용자 ID 기반의 `memberUserIds`를 사용한다.

다만 서비스 화면에서 내부 DB ID를 직접 사용하는 방식은 장기적으로 적절하지 않으므로, Phase 1.5에서 publicId 기반 사용자 검색 및 초대 방식으로 전환한다.

Phase 1의 `sourceType = MANUAL` 구조는 Phase 1.5의 친구 채팅, Phase 2의 오픈 채팅 및 AI 채팅으로 확장하기 위한 기반이다.
