# 채팅 기능 스코프 정의

## 1. 1차 개발 범위

1차 개발 목표는 `실시간 다국어 번역 채팅 MVP`이다.

### 1.1 채팅방

- 1:1 채팅방 생성
- 그룹 채팅방 생성
- 내가 참여 중인 채팅방 목록 조회
- 채팅방 상세 조회
- 그룹 채팅방 멤버 추가
- 그룹 채팅방 퇴실

### 1.2 메시지

- 텍스트 메시지 송신
- 텍스트 메시지 수신
- 메시지 원문 저장
- 최신 100개 메시지 조회
- cursor 기반 과거 메시지 100개 단위 추가 조회
- 메시지 삭제는 1차에서는 soft delete 수준으로 제한

### 1.3 번역

- 채팅방 멤버들의 언어 설정을 기준으로 번역 대상 언어 계산
- 메시지별 번역 PENDING 생성
- 비동기 번역 실행
- 번역 완료 후 WebSocket 이벤트 송신
- 번역 실패 상태 저장
- 동일 메시지 + 동일 언어 번역 중복 방지

### 1.4 언어 설정

- 채팅방 참여자별 학습 언어 설정
- 채팅방 참여자별 수신 번역 언어 설정
- 최초 참여 시 사용자 기본 언어를 적용

### 1.5 Frontend

- 채팅방 목록 화면
- 채팅방 상세 화면
- 메시지 목록 컴포넌트
- 메시지 입력 컴포넌트
- 번역 표시 컴포넌트
- 위 스크롤 과거 메시지 로딩
- WebSocket 연결/수신 처리
- 번역 중/번역 완료/번역 실패 표시

## 2. 1차 개발 제외 범위

- AI 채팅 멤버
- 오픈 채팅
- AI 자발 발언
- 방당 AI 최대 3명 관리
- 읽음 처리
- 미읽음 수 표시
- 온라인 상태 표시
- 푸시 알림
- 신고/차단 기능
- 파일/이미지 메시지
- 음성 메시지
- 대규모 Redis scale-out

## 3. 2차 개발 범위

### 3.1 오픈 채팅

- 공개 채팅방 목록
- 오픈 채팅방 생성
- 오픈 채팅방 참여/퇴장
- 공개 범위 설정
- 운영/관리 기능 검토

### 3.2 AI 채팅 멤버

- 채팅방에 AI 멤버 추가
- 방당 최대 3명 제한
- AI persona 설정
- @AI 호출 방식 응답
- AI 자발 발언 ON/OFF
- 자동 발언 쿨타임 설정

### 3.3 확장 기능

- 읽음 처리
- 미읽음 수 표시
- 온라인 상태
- Redis Pub/Sub 또는 Stream 검토
- 번역 Queue화 검토
- 알림 기능

## 4. 우선순위

| 우선순위 | 내용 |
|---|---|
| High | 채팅방, 메시지, WebSocket, 최신 100개 조회, 비동기 번역 |
| Medium | 언어 설정 UI, 번역 실패 표시, 그룹 멤버 관리 |
| Low | 메시지 삭제, 세부 UI polish, 운영 보조 기능 |
