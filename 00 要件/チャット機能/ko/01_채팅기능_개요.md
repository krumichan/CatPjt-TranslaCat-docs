# 채팅 기능 개요

## 1. 목적

TranslaCat 채팅 기능의 목적은 전 세계 사용자가 각자 자신의 언어로 대화하면서도, 상대방의 메시지를 자신이 원하는 언어로 이해할 수 있게 하는 것이다.

또한 사용자가 자신의 학습 언어를 설정하면, 자신이 작성한 메시지 하단에 학습용 번역을 표시하여 언어 학습에도 활용할 수 있도록 한다.

## 2. 핵심 컨셉

```text
사용자는 자신의 언어로 메시지를 작성한다.
다른 참여자는 각자 선택한 언어로 번역을 확인한다.
번역은 실시간에 가깝게 비동기로 표시된다.
```

## 3. 주요 사용자 경험

### 3.1 내가 메시지를 작성한 경우

사용자 A가 한국어로 메시지를 작성하고, 자신의 학습 언어를 일본어로 설정한 경우 다음과 같이 표시한다.

```text
나: 오늘은 조금 피곤해.
JA: 今日は少し疲れている。
```

### 3.2 상대방 메시지를 읽는 경우

사용자 B가 영어로 메시지를 작성하고, 사용자 A의 수신 번역 언어가 일본어인 경우 사용자 A에게는 다음과 같이 표시한다.

```text
B: I'm a little tired today.
JA: 今日は少し疲れている。
```

## 4. 기능 구분

### 1차 개발

- 사람끼리 사용하는 실시간 채팅
- 개인별 번역 언어 설정
- 비동기 번역 표시
- 채팅 이력 최신 100개 표시
- 과거 메시지 추가 로딩

### 2차 개발

- AI 채팅 멤버
- 오픈 채팅
- 읽음/미읽음/온라인 상태
- Redis 등 확장 구조 검토

## 5. 중요한 설계 원칙

- 메시지는 번역 완료를 기다리지 않고 즉시 저장/전송한다.
- 번역은 메시지 저장 이후 비동기로 처리한다.
- 번역 실패는 메시지 송수신 실패로 취급하지 않는다.
- 같은 메시지와 같은 언어의 번역은 한 번만 생성한다.
- FE는 원문 메시지와 번역 완료 이벤트를 별도로 처리한다.
