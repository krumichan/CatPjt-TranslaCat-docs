# チャット機能スコープ定義

## 1. ドキュメント目的

本ドキュメントは、TranslaCatチャット機能の Phase 1 / Phase 1.5 / Phase 2 開発範囲を定義する。

チャット機能は、基本的なメッセージ送受信だけでなく、ユーザーごとの翻訳言語設定、非同期AI翻訳、友達ベースのチャット、オープンチャット、AIチャットメンバー、オープンチャット専用仮想プロフィールまで拡張される可能性がある。

そのため、各Phaseの開発範囲を明確に区分し、Phase 1開発中の不要なスコープ拡大を防ぎ、以降のPhase 1.5 / Phase 2拡張時に既存構造を大きく作り直さないようにすることを目的とする。

## 2. 全体Phase区分

| Phase     | 区分              | 目的                                   |
| --------- | --------------- | ------------------------------------ |
| Phase 1   | 基本チャットMVP       | 人間同士で利用するリアルタイム多言語翻訳チャット機能を実装する      |
| Phase 1.5 | 友達ベースのソーシャルチャット | 友達一覧、友達追加、プロフィール、ブロック、友達チャットの基盤を構築する |
| Phase 2   | オープンチャット / AI拡張 | オープンチャット、AI参加者、仮想プロフィールなどの拡張機能を実装する  |

## 3. Phase 1 開発範囲

Phase 1は基本チャットMVPを実装する段階として定義する。

### 3.1 チャットルーム

* 1:1チャットルーム作成
* グループチャットルーム作成
* チャットルーム一覧表示
* チャットルーム詳細取得
* チャットルーム参加者数表示
* 権限のないチャットルームへのアクセス制御

### 3.2 メッセージ

* メッセージ原文保存
* メッセージ一覧取得
* 最新メッセージ初期表示
* 以前のメッセージ追加読み込み
* WebSocketによるリアルタイムメッセージ送受信
* WebSocket再接続時のRESTによる最新メッセージ同期
* メッセージ削除状態表示の基盤準備

### 3.3 翻訳

* メッセージ送信後の非同期翻訳
* メッセージ別翻訳結果保存
* 翻訳状態管理

  * PENDING
  * COMPLETED
  * FAILED
* 翻訳完了イベント受信
* 翻訳失敗イベントを受信可能な構造
* メッセージ別翻訳表示
* 翻訳が存在しない場合は原文を維持
* 翻訳失敗時は原文を維持

### 3.4 ユーザー別言語設定

* ユーザー別原文基準言語設定
* ユーザー別受信翻訳言語設定
* ユーザー別原文表示有無設定
* ユーザー別翻訳表示有無設定
* 設定変更後のメッセージ翻訳表示基準への反映

### 3.5 Phase 1.5 / Phase 2 向け構造補正

Phase 1では、将来的な拡張のために以下の構造を反映する。

* `ChatRoomSourceType` 導入

  * MANUAL
  * FRIEND
  * OPEN
  * AI
* Phase 1のチャットルーム作成では基本的に `sourceType = MANUAL` を使用する
* Phase 1.5の友達チャットは以降 `sourceType = FRIEND` として拡張する
* Phase 2のオープンチャット派生チャットは以降 `sourceType = OPEN` として拡張する
* Phase 2のAIチャットは以降 `sourceType = AI` として拡張する
* システムメッセージとAIメッセージを区別するために `senderType = SYSTEM` を導入する

## 4. Phase 1.5 開発範囲

Phase 1.5は、友達ベースのソーシャルチャット機能を構築する段階として定義する。

### 4.1 ユーザープロフィール

* 共通ユーザープロフィール
* ニックネーム
* プロフィール画像
* TranslaCat専用publicId
* publicIdによるユーザー識別
* 自分のプロフィール取得/修正

### 4.2 友達機能

* publicIdベースのユーザー検索
* 友達リクエスト
* 友達リクエスト承認
* 友達リクエスト拒否
* 友達一覧取得
* 友達削除

### 4.3 ブロック基礎機能

* ユーザーブロック
* ブロック解除
* ブロックしたユーザーからの友達リクエスト制限
* ブロックしたユーザーからのグループ招待制限

### 4.4 友達1:1チャット

* 友達一覧から1:1チャット開始
* 既存の友達1:1チャットルーム再利用
* 友達1:1チャットルームの重複生成防止

### 4.5 友達グループチャット

* 友達一覧ベースのグループチャット作成
* 友達一覧ベースのグループ招待
* 非友達招待時の承認制参加検討

## 5. Phase 2 開発範囲

Phase 2は、オープンチャットおよびAI参加者を含む拡張段階として定義する。

### 5.1 オープンチャット

* オープン1:1チャット
* オープングループチャット
* オープンチャットルーム一覧
* オープンチャットルーム検索
* オープンチャットルーム最大人数設定
* オープンチャットルーム入室ポリシー
* オープンチャットルーム運用ポリシー

### 5.2 AIチャット

* AIとの1:1チャット
* オープングループチャットへのAI招待
* オープングループチャット内AI最大N名制限
* AIメッセージとユーザーメッセージの区別
* AI応答制御

### 5.3 AI参加ポリシー

オープンチャットルームは、ルーム単位でAI参加ポリシーを持つ。

* AI参加不可
* AI参加可能 / AI表示
* AI参加可能 / 個別AI表示なし

AIが存在する可能性はルーム情報に表示し、個別AI表示有無はルームポリシーに従って統一する。

### 5.4 オープンチャット専用仮想プロフィール

* オープンチャットルーム別仮想プロフィール作成
* 基本プロフィールまたは仮想プロフィールを選択して入室
* オープンチャットルーム別ニックネーム設定
* オープンチャットルーム別プロフィール画像設定
* ルーム退出時の仮想プロフィール非活性化検討

### 5.5 運用および拡張機能

* 通報
* ブロック高度化
* 強制退室
* 運営者/管理者ロール
* 既読/未読管理
* オンライン状態表示
* Redisベースのscale-out検討

## 6. Phase 1 対象外範囲

以下の機能はPhase 1開発範囲から除外する。

* 友達機能
* 友達リクエスト/承認/拒否
* ユーザープロフィール詳細管理
* publicIdベースのユーザー検索
* ブロック機能
* オープンチャット
* AIチャットメンバー
* AI自発発言
* オープンチャット専用仮想プロフィール
* 通報/運用ポリシー
* 既読/未読管理
* オンライン状態表示
* Redisベースのscale-out構成

## 7. Phase 1 完了基準

Phase 1は以下の条件を満たした場合に完了と判断する。

* チャットルーム一覧を取得できる
* チャットルームを作成できる
* チャットルーム参加者数が表示される
* チャットルーム詳細画面へ遷移できる
* 権限のないチャットルームへのアクセスが制御される
* メッセージを送信できる
* WebSocketでメッセージを受信できる
* 初期メッセージ一覧を取得できる
* 以前のメッセージを追加読み込みできる
* メッセージ翻訳結果を表示できる
* 翻訳完了イベントを受信して画面に反映できる
* 翻訳失敗状態を表示できる
* ユーザー別言語設定を変更できる
* 言語設定変更後に翻訳表示基準が反映される
* FEアプリケーションが正常にビルドされる
* BEアプリケーションが正常にビルドされる

## 8. 備考

Phase 1のチャットルーム作成リクエストは、内部ユーザーIDベースの `memberUserIds` を使用する。

ただし、サービス画面で内部DB IDを直接使用する方式は長期的には適切ではないため、Phase 1.5でpublicIdベースのユーザー検索および招待方式へ移行する。

Phase 1の `sourceType = MANUAL` 構造は、Phase 1.5の友達チャット、Phase 2のオープンチャットおよびAIチャットへ拡張するための基盤である。
