# チャット機能 概要

## 1. 目的

TranslaCatチャット機能の目的は、世界中のユーザーが各自の言語で会話しながら、相手のメッセージを自分が希望する言語で理解できるようにすることである。

また、ユーザーが学習言語を設定した場合、自分が作成したメッセージ下部に学習用翻訳を表示し、言語学習にも活用できるようにする。

## 2. コアコンセプト

```text
ユーザーは自分の言語でメッセージを入力する。
他の参加者は各自選択した言語で翻訳を確認する。
翻訳はリアルタイムに近い形で非同期表示される。
```

## 3. 主なユーザー体験

### 3.1 自分がメッセージを作成した場合

ユーザーAが韓国語でメッセージを作成し、自分の学習言語を日本語に設定した場合、以下のように表示する。

```text
自分: 오늘은 조금 피곤해.
JA: 今日は少し疲れている。
```

### 3.2 相手のメッセージを読む場合

ユーザーBが英語でメッセージを作成し、ユーザーAの受信翻訳言語が日本語の場合、ユーザーAには以下のように表示する。

```text
B: I'm a little tired today.
JA: 今日は少し疲れている。
```

## 4. 機能区分

### 1次開発

- 人間同士のリアルタイムチャット
- 個人別翻訳言語設定
- 非同期翻訳表示
- チャット履歴最新100件表示
- 過去メッセージ追加読み込み

### 2次開発

- AIチャットメンバー
- オープンチャット
- 既読/未読/オンライン状態
- Redis等の拡張構成検討

## 5. 重要な設計原則

- メッセージは翻訳完了を待たずに即時保存・送信する。
- 翻訳はメッセージ保存後に非同期で処理する。
- 翻訳失敗はメッセージ送受信失敗として扱わない。
- 同一メッセージ・同一言語の翻訳は1回のみ作成する。
- FEは原文メッセージと翻訳完了イベントを別々に処理する。
