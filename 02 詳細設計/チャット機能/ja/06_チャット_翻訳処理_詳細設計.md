# チャット 翻訳処理 詳細設計

## 1. 文書目的

本書は、TranslaCatチャット機能1次MVPでメッセージ翻訳を非同期処理する詳細フローを定義する。

1次目標は、原文メッセージ送受信を妨げず、翻訳結果を後から画面に反映することである。

---

## 2. 基本原則

- 原文メッセージは即時保存し、即時WebSocket配信する。
- 翻訳は非同期で処理する。
- 翻訳失敗がメッセージ送受信を妨げない。
- 翻訳状態はDBに保存する。
- 翻訳完了/失敗はWebSocketイベントで配信する。

---

## 3. 処理フロー

```text
1. ユーザーがメッセージ送信
2. BEがチャットルームメンバー権限検証
3. BEがchat_messageに原文保存
4. BEが翻訳対象言語計算
5. BEがchat_message_translation PENDING作成
6. BEがchat.message.created配信
7. BEがchat.translation.pending配信
8. BEがAI Serverへ翻訳リクエスト
9. AI Serverが翻訳結果返却
10. BEが翻訳結果保存
11. BEがchat.translation.completed配信
```

失敗時:

```text
AI Serverリクエスト失敗または翻訳失敗
→ chat_message_translation.status = FAILED
→ failure_reason保存
→ chat.translation.failed配信
```

---

## 4. 翻訳対象言語計算

初期方針:

```text
チャットルームactive memberのtranslationLanguageCodeを収集
送信者を除外
重複除去
null除去
```

例:

| ユーザー | 役割 | 翻訳言語 |
|---|---|---|
| A | 送信者 | ja |
| B | 受信者 | ko |
| C | 受信者 | en |
| D | 受信者 | ko |

対象言語:

```text
ko, en
```

---

## 5. 翻訳状態

```text
PENDING
COMPLETED
FAILED
```

---

## 6. AI Serverリクエスト候補

```text
POST /api/v1/chat/translate
```

Request:

```json
{
  "sourceLanguageCode": "ja",
  "targetLanguageCodes": ["ko", "en"],
  "text": "今日何食べる？",
  "context": "chat"
}
```

Response:

```json
{
  "translations": [
    {
      "languageCode": "ko",
      "text": "오늘 뭐 먹을래?"
    }
  ]
}
```

---

## 7. prompt基準

- 短いチャット口調を維持する。
- 原文の意味を変えない。
- 絵文字を維持する。
- URLを維持する。
- `@mention`を維持する。
- 金額、日付、時間、固有名詞を維持する。
- 説明を付けない。
- 翻訳文のみ返却する。

---

## 8. BE非同期処理方式

1次候補:

```text
Spring @Async
ApplicationEvent + @TransactionalEventListener(AFTER_COMMIT)
```

推奨フロー:

```text
メッセージ保存transaction commit
→ AFTER_COMMIT event
→ 翻訳非同期リクエスト
→ 翻訳結果保存transaction
→ WebSocket翻訳完了イベント配信
```

Redis Queueまたは外部message queueは1次範囲外とする。

---

## 9. 失敗処理

失敗タイプ候補:

```text
AI_SERVER_TIMEOUT
AI_SERVER_ERROR
INVALID_TRANSLATION_RESPONSE
TRANSLATION_EMPTY
UNKNOWN_ERROR
```

処理基準:

- 失敗した言語のみFAILED処理する。
- 一部言語成功、一部言語失敗を許可する。
- 失敗時も原文メッセージは維持する。
