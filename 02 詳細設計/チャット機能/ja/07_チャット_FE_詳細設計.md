# チャット FE 詳細設計

## 1. 文書目的

本書は、TranslaCatチャット機能1次MVPのFrontend詳細設計を定義する。

1次目標は、ユーザーがチャットルーム一覧を確認し、チャットルームに入室してメッセージを送受信し、翻訳状態と結果を確認できるUIを実装することである。

## 3. 画面構成

### 3.1 チャットルーム一覧

Route候補:

```text
/chat
```

機能:

- 自分のチャットルーム一覧取得
- 最後のメッセージ表示
- チャットルーム作成モーダル表示
- チャットルーム詳細遷移

### 3.2 チャットルーム詳細

Route候補:

```text
/chat/{roomId}
```

機能:

- 最新メッセージ100件取得
- WebSocket接続
- メッセージ送信
- メッセージ受信
- 翻訳状態表示
- 過去メッセージ追加取得
- 言語設定UI表示

## 5. WebSocket処理基準

1次初期実装ではoptimistic updateなしで、サーバーイベント基準でメッセージを追加する。

```text
ユーザーがメッセージ入力
→ WebSocket send
→ サーバー chat.message.created 受信
→ メッセージ一覧に追加
→ chat.translation.pending 受信
→ 翻訳中表示
→ chat.translation.completed 受信
→ 既存メッセージのtranslations更新
```

重複防止基準:

```text
message.id
```

---

## 6. メッセージ表示方針

メッセージbubbleに表示する項目:

- 送信者名
- 原文content
- 作成時刻
- 翻訳状態
- 翻訳結果

翻訳状態表示:

| 状態 | 表示 |
|---|---|
| PENDING | 翻訳中... |
| COMPLETED | 翻訳文表示 |
| FAILED | 翻訳失敗または表示省略 |

---

## 7. 過去メッセージ取得

初期入室:

```text
最新100件取得
```

上スクロール:

```text
cursorMessageId基準で以前100件取得
```

注意点:

- 重複メッセージ除去
- スクロール位置維持
- loading中の重複リクエスト防止
- hasNext false処理
