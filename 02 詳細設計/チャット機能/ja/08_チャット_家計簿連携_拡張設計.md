# チャット家計簿連携拡張設計

## 1. 目的

本書は将来的にチャットルームと家計簿を連携するための拡張設計方針を整理する。

## 2. 基本方針

1次開発では家計簿連携を実装しない。

ただし、将来的にグループチャットルームまたはオープンチャットルームに1つ以上の家計簿を連携できるよう、チャットドメインと家計簿ドメインを強く結合しない。

## 3. 連携テーブル候補

```text
chat_room_account_book_link
```

## 4. 設計原則

- chat_roomにaccountBookIdを直接持たせない。
- チャットルーム権限と家計簿権限は分離する。
- 家計簿アクセス権限はaccount_book_memberを基準に判断する。

## 5. 詳細内容

3次以降の開発時に補強する。
