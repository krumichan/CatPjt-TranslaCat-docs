# 채팅 가계부 연동 확장설계

## 1. 문서 목적

본 문서는 TranslaCat 채팅 기능과 가계부 기능의 향후 연동 가능성을 정리한다.

가계부 연동은 1차 채팅 MVP 범위가 아니며, 2차 또는 3차 이후 확장 후보로 관리한다.
다만 1차 채팅 도메인 설계에서 가계부 연동을 방해하는 강한 결합을 만들지 않기 위해 본 문서에 확장 방향을 남긴다.

---

## 2. 연동 목적

채팅방과 가계부를 연동하면 다음 사용 사례를 지원할 수 있다.

- 친구 그룹 채팅방의 여행 정산
- 동호회 오픈 채팅방의 회비 관리
- 모임별 공동 지출 기록
- 회식비/숙소비/교통비 정산
- 영수증 분석 결과를 채팅방에 공유
- 채팅 메시지 기반 지출 등록 후보 생성

---

## 3. 설계 원칙

가장 중요한 원칙은 다음과 같다.

```text
채팅방과 가계부를 직접 결합하지 않는다.
```

금지 방향:

```text
chat_room.account_book_id
```

이 방식은 하나의 채팅방에 하나의 가계부만 연결할 수 있게 만들어 향후 확장에 불리하다.

권장 방향:

```text
chat_room_account_book_link
```

---

## 4. 관계 설계

```text
chat_room 1 ─ N chat_room_account_book_link N ─ 1 account_book
```

이 구조로 다음 요구를 만족할 수 있다.

- 하나의 채팅방에 여러 가계부 연결
- 하나의 가계부를 여러 채팅방에 공유
- 연결 상태 soft delete
- 연결 순서 관리
- 연동한 사용자 기록

---

## 5. 연결 테이블 후보

### chat_room_account_book_link

| 컬럼 | 설명 |
|---|---|
| id | PK |
| chat_room_id | 채팅방 ID |
| account_book_id | 가계부 ID |
| linked_by_user_id | 연동한 사용자 ID |
| link_status | ACTIVE, REMOVED |
| display_order | 채팅방 내 표시 순서 |
| created_at | 생성일시 |
| deleted_at | 삭제일시 |

unique 후보:

```sql
CREATE UNIQUE INDEX uk_chat_room_account_book_link
ON chat_room_account_book_link(chat_room_id, account_book_id);
```

---

## 6. 권한 원칙

채팅 권한과 가계부 권한은 분리한다.

```text
채팅방 멤버라고 해서 가계부 멤버가 되는 것은 아니다.
가계부 멤버라고 해서 채팅방 멤버가 되는 것도 아니다.
```

가계부 접근 시에는 반드시 기존 가계부 권한을 확인한다.

```text
account_book_member
```

---

## 7. 후속 기능 후보

### 7.1 가계부 연동

채팅방 설정에서 가계부를 연결한다.

### 7.2 가계부 초대 연계

가계부를 채팅방에 연결할 때 채팅방 멤버를 가계부 멤버로 초대한다.

### 7.3 정산 결과 공유

가계부 월별/항목별 정산 결과를 채팅 메시지로 공유한다.

### 7.4 영수증 분석 공유

AI 영수증 분석 결과를 특정 채팅방에 공유한다.

### 7.5 채팅 기반 거래 후보 생성

예시:

```text
A: 오늘 회식비 12000엔 내가 냈어
B: 그럼 4명이서 나누자
```

AI가 이 메시지를 바탕으로 거래 등록 후보를 제안한다.

## 8. 개발 단계 제안

```text
1차: 채팅 MVP 구현, 가계부 연동 미구현
2차: 오픈 채팅/AI/읽음/알림/Redis
3차: 채팅방-가계부 연동 테이블 도입
4차: 정산/AI 지출 후보/영수증 공유 확장
```

---

## 9. 1차 설계에서 지켜야 할 것

- `chat_room`에 `account_book_id`를 넣지 않는다.
- 채팅방 멤버 권한과 가계부 멤버 권한을 섞지 않는다.
- 채팅 메시지 도메인이 가계부 거래 도메인을 직접 의존하지 않도록 한다.
- 향후 시스템 메시지로 가계부 이벤트를 표시할 수 있도록 `ChatMessageType.SYSTEM`을 열어둔다.
- 향후 링크 카드 형태 UI를 표시할 수 있도록 message metadata 확장 가능성을 고려한다.
