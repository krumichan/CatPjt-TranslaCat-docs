# 채팅 FE 상세설계

## 1. 문서 목적

본 문서는 TranslaCat 채팅 기능 1차 MVP의 Frontend 상세설계를 정의한다.

1차 목표는 사용자가 채팅방 목록을 보고, 채팅방에 입장하여 메시지를 주고받고, 번역 상태와 결과를 확인할 수 있는 UI를 구현하는 것이다.

## 3. 화면 구조

### 3.1 채팅방 목록

Route 후보:

```text
/chat
```

기능:

- 내 채팅방 목록 조회
- 마지막 메시지 표시
- 채팅방 생성 모달 진입
- 채팅방 상세 이동

---

### 3.2 채팅방 상세

Route 후보:

```text
/chat/{roomId}
```

기능:

- 최신 메시지 100개 조회
- WebSocket 연결
- 메시지 송신
- 메시지 수신
- 번역 상태 표시
- 과거 메시지 추가 로딩
- 언어 설정 UI 진입

---

### 3.3 채팅방 생성 모달

기능:

- 방 이름 입력
- 멤버 선택 또는 userId/publicId 입력
- 방 생성 API 호출
- 생성 후 상세 화면 이동


---

## 5. WebSocket 처리 기준

1차 초기 구현에서는 optimistic update 없이 서버 이벤트 기준으로 메시지를 추가한다.

처리 흐름:

```text
사용자가 메시지 입력
→ WebSocket send
→ 서버 chat.message.created 수신
→ 메시지 목록에 추가
→ chat.translation.pending 수신
→ 번역 중 표시
→ chat.translation.completed 수신
→ 기존 메시지의 translations 갱신
```

중복 방지 기준:

```text
message.id
```

---

## 6. 메시지 표시 정책

메시지 bubble에는 다음을 표시한다.

- 발신자 이름
- 원문 content
- 생성 시각
- 번역 상태
- 번역 결과

번역 상태 표시:

| 상태 | 표시 |
|---|---|
| PENDING | 번역 중... |
| COMPLETED | 번역문 표시 |
| FAILED | 번역 실패 또는 표시 생략 |

---

## 7. 과거 메시지 로딩

초기 진입:

```text
최신 100개 조회
```

위로 스크롤:

```text
cursorMessageId 기준 이전 100개 조회
```

주의사항:

- 중복 메시지 제거
- 스크롤 위치 유지
- 로딩 중 중복 요청 방지
- hasNext false 처리

---

## 8. 언어 설정 UI

채팅방별 내 언어 설정을 변경한다.

입력 후보:

- originalLanguageCode
- translationLanguageCode
- showOriginal
- showTranslation

초기값은 사용자 locale 또는 기본 언어 설정을 사용한다.

---

## 9. 1차 구현 우선순위

```text
1. chat type 정의
2. chatApi.ts 구현
3. /chat 목록 화면
4. 채팅방 생성 모달
5. /chat/{roomId} 기본 레이아웃
6. 메시지 목록 조회
7. WebSocket 연결
8. 메시지 송수신
9. 번역 상태/결과 표시
10. 과거 메시지 추가 로딩
11. 언어 설정 UI
```
